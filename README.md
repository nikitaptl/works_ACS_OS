# Потылицин Никита Сергеевич, БПИ228
## Работа была выполнена на 10 баллов
### Программа на 8 баллов
Работа процессов завершается после нажатия пользователем Ctrl + \ - это вызывает сигнал SIGQUIT в процессе сервера, и он помещает в разделяемую память число -1, удаляет разделяемую память, завершает работу. Клиент, получив число -1, так же завершает свою работу.
### Программа на 9 баллов
Первым запускается Клиент. Он создаёт разделяемую память и передаёт через неё свой pid серверу, ожидает начала генерации чисел. Сервер, получив pid клиента, начинает генерацию чисел. По тому же принципу, после нажатия Ctrl + \ сервер получает сигнал SIGQUIT и с помощью kill() отправляет SIGQUIT клиенту, удаляет разделяемую память, завершает работу. Клиент, получив сигнал, так же завершает свою работу.
### Программа на 10 баллов
То же самое, что и в прошлой программе, но клиент получает сигнал SIGKILL. После этого клиент не может выполнить никаких дополнительных действий и принудительно завершает работу, поэтому работу с удалением участков разделямой памяти полностью берёт на себя сервер.
